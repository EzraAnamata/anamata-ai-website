# anamata.ai

The AI-operated marketing site for [Anamata](https://anamata.eu). Anna — the AI
employee — is the product; every page also carries the live, typeset **operating
record** of the AI team that runs the site. The site is its own evidence.

Built with [Astro](https://astro.build) as a static site. Design is governed by
`design-tokens.json` (tokens are law → generated CSS). AI-authored changes are
committed under the `Otto <otto@anamata.ai>` identity — see
[`docs/authorship.md`](docs/authorship.md) and [`CLAUDE.md`](CLAUDE.md).

## Develop

```bash
npm install
npm run dev        # astro dev server (default http://localhost:4321)
npm run build      # regenerates tokens CSS + builds to dist/
npm test           # build + run the launch checks (vitest)
npm run test:only  # run the launch checks against the current dist/ (no rebuild)
```

`npm run dev`/`npm run build` first run `npm run tokens`
(`scripts/generate-tokens-css.mjs`), which regenerates
`src/styles/tokens.generated.css` from `design-tokens.json`. That generated file
is git-ignored — never hand-edit it; edit the tokens.

### Generated assets (committed, not part of the build)

Some binary assets are generated once by a script and committed as the shipped
artifact (they are **not** regenerated by `npm run build`):

- **Hero frames** — `public/anna/frame-00..32.webp` +
  `anna-split-face.webp`, via `scripts/build-hero-frames.mjs`. Source GIF/PNG
  live outside the repo and are never committed; all shipped assets are
  metadata-stripped.
- **Social card** — `public/og-image.png` (1200×630), via `npm run og`
  (`scripts/build-og-image.mjs`). Composed from the in-repo, metadata-stripped
  split-face portrait plus the outlined wordmark; wired as the default
  `og:image` (absolute URL) in `src/layouts/Base.astro`. Brand type (Poppins
  200/600, IBM Plex Mono 600) is rendered as real glyph **outlines** via
  opentype.js reading the `@fontsource` WOFF files — librsvg silently ignores
  data-URI `@font-face`, so paths are the only reliable mechanism; the script
  hard-fails if any glyph path is malformed. Pure Node, no browser, no committed
  font binaries, no network. No imagery is ever generated via Higgsfield without
  explicit approval.

## Launch checks

`tests/launch-checks.test.mjs` runs against the built `dist/` and defines "done":
pages exist, the Art. 50 (EU AI Act) transparency notice and machine-readable
AI-provenance meta are on every page, the operating-record ledger is **real**
(derived from this repo's git history, never fabricated) and crawlable, no
request leaves the origin (self-hosted fonts, GDPR), semantic-HTML and
contrast/focus a11y basics hold, the design tokens match the built CSS, and the
deploy workflow only ships through the human-approved `production` environment.

## Deploy

Every change ships via **PR → human merge → gated deploy**. Pushing to `main`
triggers `.github/workflows/deploy.yml`: build + launch checks, then a `deploy`
job bound to the `production` environment (a named human must approve it in the
GitHub UI). The approval becomes part of the site's own operating record. The
deploy step rsyncs `dist/` to the Anamata cloud server, which serves the static
site via [Caddy](https://caddyserver.com). Never push to `main`; never bypass the
gate.

### Route redirects (`/anna`, `/approach` → `/about`)

`/anna` and `/approach` were folded into `/about`. The static build emits
meta-refresh redirect stubs for both (`astro.config.mjs` `redirects`), which are
live today. True HTTP 301s are available as a one-time, manual Caddyfile edit —
see [`deploy/redirects.caddy`](deploy/redirects.caddy). That is a server-config
change outside the `dist/`-only deploy pipeline (a named, human-gated action), so
it is documented rather than automated; the meta-refresh stubs remain the
accepted deployed behaviour until it is applied. Both approaches send visitors
and search engines to `/about`.
